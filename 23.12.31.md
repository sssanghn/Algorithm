```
문제 2 Team
어느 날 세 명의 절친한 친구 페트야, 바스야, 토냐는 팀을 구성하여 프로그래밍 대회에 참가하기로 결정했습니다. 참가자들은 보통 프로그래밍 콘테스트에서 몇 가지 문제를 풀게 됩니다. 시작하기 훨씬 전에 친구들은 적어도 두 명 이상이 해결책에 대해 확신하면 문제를 구현하기로 결정했습니다. 그렇지 않으면 친구들은 문제 해결책을 작성하지 않습니다.

이 콘테스트는 참가자들에게 n개의 문제를 제공합니다. 우리가 알고 있는 각 문제에 대해 어떤 친구가 해결책을 확신하는지 알려주세요. 친구들이 풀이를 작성할 문제 수를 찾도록 도와주세요.

입력
첫 번째 입력 줄에는 콘테스트의 문제 수인 단일 정수 n(1 ≤ n ≤ 1000)이 들어 있습니다. 그런 다음 n 줄에는 각각 세 개의 정수가 포함되며, 각 정수는 0 또는 1입니다. 줄의 첫 번째 숫자가 1이면 페트야는 문제의 해답을 확신하는 것이고, 그렇지 않으면 확신하지 못하는 것입니다. 두 번째 숫자는 해결책에 대한 바샤의 견해를, 세 번째 숫자는 토냐의 견해를 나타냅니다. 줄의 숫자는 공백으로 구분되어 있습니다.

출력
친구들이 콘테스트에서 구현할 문제 수인 정수를 하나 출력합니다.

예시 입력
1 1 0
0 1 1
1 1 0
예시 출력
3
```
```
sols = 0
for _ in range(int(input())):
	sols += sum(int(i) for i in input().split()) >= 2
print(sols)
```

```
문제 3 Soldier and Bananas
한 병사가 상점에서 바나나 w개를 사고 싶어합니다. 그는 첫 번째 바나나에 k달러, 두 번째 바나나에 2k달러를 지불해야 합니다(다시 말해, 그는 i번째 바나나에 i*k달러를 지불해야 합니다).

그는 n달러를 가지고 있습니다. 그가 바나나 W개를 사기 위해 친구 병사에게 몇 달러를 빌려야 할까요?

입력
첫 번째 줄에는 세 개의 양의 정수 k, n, w(1 ≤ k, w ≤ 1000, 0 ≤ n ≤ 109), 첫 번째 바나나의 가격, 병사가 가진 초기 달러 수, 원하는 바나나 수가 주어집니다.

출력
병사가 친구에게 빌려야 하는 달러의 양을 하나의 정수로 출력합니다. 돈을 빌릴 필요가 없으면 0을 출력합니다.

예시 입력
3 17 4
예시 출력
13
```
```
first, money, num = map(int, input().split())
cost = 0

for i in range(num):
	cost += first * (i + 1)

print(max(0, cost - money))
# if cost > money:
	  print(cost - money)
  else:
	  print(0)
```

```
문제 4 Promotion Counting
소 베시는 농부 존이 소에 대한 상식을 겨루는 온라인 경연대회인 미국 소 올림피아드(USACO)의 진행을 돕고 있습니다.
농부 존은 최근 참가자의 다양한 배경을 반영하여 콘테스트를 브론즈, 실버, 골드, 플래티넘의 네 가지 난이도 부문으로 확장했습니다. 모든 신규 참가자는 브론즈 디비전부터 시작하며, 콘테스트에서 만점을 받으면 한 단계 높은 디비전으로 승격됩니다. 심지어 한 참가자가 같은 콘테스트에서 여러 번 승급할 수도 있습니다. 농부 존은 모든 콘테스트 참가자의 목록과 현재 디비전을 추적하여 콘테스트를 개최할 때마다 모든 참가자를 적절한 수준에서 시작할 수 있도록 합니다.

가장 최근 콘테스트의 결과를 게시할 때 농부 존은 브론즈에서 실버로, 실버에서 골드로, 골드에서 플래티넘으로 승급한 참가자 수에 대한 정보를 포함하려고 합니다. 하지만 그는 콘테스트 기간 중에 발생한 승급을 계산하는 것을 소홀히 했습니다. 하지만 영리한 소인 베시는 농부 존이 콘테스트 전후의 각 레벨 참가자 수만으로 승급자 수를 추론할 수 있다는 사실을 알아챕니다. 농부가 이 계산을 할 수 있도록 도와주세요!

입력 형식(파일 promote.in):

입력은 네 줄로 구성되며, 각 줄은 0...1,000,000 범위의 정수 두 개를 포함합니다. 첫 번째 줄은 콘테스트 전후에 등록된 브론즈 참가자 수를 지정합니다. 두 번째 줄은 콘테스트 전후의 실버 참가자 수를 지정합니다. 세 번째 줄은 콘테스트 전후의 골드 참가자 수를 나타냅니다. 마지막 줄은 콘테스트 전후의 플래티넘 참가자 수를 지정합니다.

출력 형식(promot.out 파일):

각각 하나의 정수를 포함하는 세 줄을 출력하세요. 첫 번째 줄에는 브론즈에서 실버로 승급한 참가자 수가 포함되어야 합니다. 두 번째 줄에는 실버에서 골드로 승급한 참가자 수가 포함되어야 합니다. 마지막 줄에는 골드에서 플래티넘으로 승급한 참가자 수가 포함되어야 합니다.

예시 입력:
1 2
1 1
1 1
1 2
예시 출력:
1
1
1
이 예에서는 콘테스트 전에 각 부문에 참가자 1명이 등록되었습니다. 콘테스트가 끝났을 때 2명의 참가자가 브론즈와 플래티넘에 등록되었습니다. 이런 일이 일어날 수 있는 한 가지 방법은 콘테스트 기간 동안 2명의 새로운 참가자가 합류하여 한 명은 플래티넘으로 승격하고 다른 한 명은 브론즈에 머물렀기 때문입니다.
```

```
import sys

sys.stdin = open("promote.in", "r")
sys.stdout = open("promote.out", "a")
```

```
문제 5 Do You Know Your ABCs?
농부 존의 소들은 매일 화상 회의 플랫폼인 "mooZ"에서 온라인 모임을 갖고 있습니다. 이들은 회의 중 재미를 더하기 위해 간단한 숫자 게임을 고안해냈습니다.
Elsie는 세 개의 양의 정수 A, B, C(A≤B≤C)를 가지고 있습니다. 이 정수는 비밀로 해야 하므로 언니인 베시에게 직접 알려주지 않습니다. 대신, Elsie는 1...109 범위에 있는 일곱 개의 정수(반드시 구별되는 것은 아님)를 베시에게 주면서 순서대로 A, B, C, A+B, B+C, C+A, A+B+C라고 주장합니다.

이 일곱 개의 숫자 목록이 주어졌을 때, 베시가 A, B, C를 결정할 수 있도록 도와주세요.

입력 형식(입력은 터미널/STDIN에서 도착합니다):

입력의 유일한 줄은 공백으로 구분된 7개의 정수로 구성됩니다.
출력 형식(터미널/스탠다드 출력으로 출력):

A, B, C를 공백으로 구분하여 인쇄합니다.

입력 예시:
2 2 11 4 9 7 9
출력 예시:
2 2 7
점수:
테스트 사례 2-3은 C≤50을 만족합니다.
테스트 사례 4~10은 추가 제약 조건을 충족하지 않습니다.
```
